## 摘要
基于大语言模型（LLM）的网络搜索智能体正将信息检索从简单的事实性问题回答转变为复杂的深度与广度搜索及研究型任务。单个ReAct风格智能体受限于单一长轨迹和有限上下文，难以同时兼顾深度与覆盖范围。现有的多智能体系统通过并行执行与聚合提升搜索覆盖，但在递归深度、协作适应性和基于证据的扩展方面仍存在明显局限。我们提出WebSwarm，一种渐进式递归委托框架，在推理过程中联合构建任务分解、递归扩展和智能体协作。WebSwarm动态实例化智能搜索节点，每个节点将局部目标与搜索模式耦合，该模式指定节点应如何组织搜索与协作。每个节点可自行求解其目标，或进一步委托子节点；求解后，将证据和结果向上返回，使父节点能进一步扩展、修订或聚合搜索过程。为引导此过程，WebSwarm首先探测与任务相关的信息在网页上的组织方式，以支撑后续节点扩展，并在同质兄弟节点间复用过程级经验。在BrowseComp-Plus、WideSearch、DeepWideSearch和GISA上的实验表明，WebSwarm在深度、广度以及交织的深度与广度任务上持续优于单智能体和多智能体基线。进一步的消融分析、任务难度、网络工具效率及模型泛化性分析解释了WebSwarm的有效性，并为多智能体搜索系统提供了见解。

## Abstract
Large language model (LLM)-based web search agents are transforming information seeking from simple factoid question answering into complex, deep-and-wide search and research-oriented tasks. A single ReAct-style agent is constrained by one long trajectory and limited context, making it difficult to handle depth and coverage simultaneously. Existing multi-agent systems improve search coverage through parallel execution and aggregation, but still exhibit clear limitations in recursive depth, collaboration adaptability, and evidence-grounded expansion. We propose WebSwarm, a progressive recursive delegation framework that jointly constructs task decomposition, recursive expansion, and agent collaboration during inference. WebSwarm dynamically instantiates agentic search nodes, each coupling a local objective with a search mode that specifies how the node should organize search and collaboration. Each node can either solve its objective itself or further delegate child nodes; after solving, it returns evidence and results upward, enabling parent nodes to further expand, revise, or aggregate the search process. To guide this process, WebSwarm first probes how task-relevant information is organized on the web to ground subsequent node expansion, and reuses process-level experience across homogeneous sibling nodes. Experiments on BrowseComp-Plus, WideSearch, DeepWideSearch, and GISA show that WebSwarm consistently outperforms single-agent and multi-agent baselines on deep, wide, and interleaved deep-and-wide tasks. Further analyses of ablation, task difficulty, web tool efficiency, and model generalization explain WebSwarm's effectiveness and provide insights for multi-agent search systems.

## 1. 核心问题与研究动机
- **背景**：基于大语言模型（LLM）的网页搜索代理正从简单事实问答向复杂深度/广度搜索及研究型任务转变。单个 ReAct 风格代理受限于单一长轨迹和有限上下文，难以同时处理深度（多跳推理）和广度（多实体覆盖）。
- **问题**：现有多智能体系统通过并行执行和聚合提升覆盖，但存在三大局限：  
  - **浅层递归深度**：任务树只能扁平分解，深层结构被迫压入单一子代理的长轨迹中。  
  - **协作适应性不足**：全局采用固定协作模式（如并行、串行），无法适应不同子任务的需求（事实查找、广域枚举、多约束验证等）。  
  - **缺乏证据驱动的扩展**：分解仅基于查询表面语义，未与网页信息的实际组织方式对齐，导致冗余检索和低召回。
- **整体目标**：提出一种能在推理过程中**渐进式、递归地**构建任务分解、扩展和协作的多智能体框架，以同时支持深度和广度搜索。

## 2. 方法论：WebSwarm
- **核心思想**：将网络搜索建模为**递归委托过程**。动态实例化智能搜索节点，每个节点耦合一个局部目标和一个搜索模式（`mode`）。节点可自行求解或进一步委托子节点，完成后向上返回证据和结果，驱动父节点扩展、修订或聚合。
- **关键技术细节**：
  - **递归委托树**：根节点接收原始任务，动态生成子节点（`(local objective, search mode)` 对）。节点可递归委托，形成层次结构。
  - **四种搜索模式**（`M = {atom, deep, wide, entity_collect}`）：
    - `atom`：原子事实查找，直接调用网页工具。
    - `deep`：迭代搜索与验证，适用于未知目标识别；内部串行调度搜索者（Searcher）和验证者（Verifier）。
    - `wide`：并行分治，适用于对一组对象收集相似信息；并行调度子节点（可为 `deep`, `entity_collect`, `wide`），合并结果。
    - `entity_collect`：多路径召回与验证，适用于未知集合枚举；并行多路原子搜索，再合并去重验证。
  - **证据更新与继续**：子节点返回结果后，父节点决定是否继续委托、修订方向或聚合返回。
  - **指导信号**：
    - **网页结构探测**（Web-Probing Agent）：执行轻量搜索，判断信息是集中在一个聚合页还是分散在多个维度（如按品牌、年份），从而指导扩展轴。
    - **兄弟节点经验复用**：在执行一批同质子任务时，先执行少量侦察节点，提取过程级经验（有效查询模式、可靠来源、失败路径等），注入后续子节点以提升效率和质量。
- **算法流程**（伪代码描述）：
  1. 创建根节点 `v0`，调用 `SolveNode(v0, ∅)`。
  2. `SolveNode` 循环：若未就绪，则进行 Web 探测（如需要），然后决定是直接求解还是委托子节点。
  3. 若委托，生成子委托集；对单个子节点调用 `SolveNode` 递归求解。
  4. 子节点返回后更新当前结果集；如需经验复用，执行侦察子集，提取经验，再执行剩余子节点。
  5. 循环直到就绪，聚合结果向上返回。

## 3. 实验设计
- **数据集/Benchmark**：
  - **BrowseComp-Plus**（深度搜索）：200 个样本，指标为答案准确率（ACC）。
  - **WideSearch-EN**（广度结构化收集）：100 个英文样本，指标为 Item-F1、Row-F1、Success Rate（SR）。
  - **DeepWideSearch-EN**（深度广度嵌套）：76 个英文样本，指标同 WideSearch。
  - **GISA**（通用信息寻求）：373 个样本，包括多项、集合、列表、表格型指标。
- **对比方法**：
  - 单智能体：ReAct（使用不同骨干：GLM-4.5、Qwen3-32B、Qwen3-235B、Kimi-K2、Qwen3.5-35B）。
  - 多智能体：Swarm-Agent、Flash-Searcher、Table-as-Search、InfoSeeker、ROMA（均使用 GLM-4.5 骨干）。
- **实现细节**：
  - 主要骨干模型为 GLM-4.5，部分实验使用 Qwen3-32B 和 Qwen3.5-35B。
  - 网页工具：Serper API（搜索）+ Jina Reader API（页面获取，上限 80K tokens）。
  - 推理引擎：SGLang，参数：温度 0.6，top-p 0.95，top-k 20，重复惩罚 1.0，存在惩罚 1.5，最大生成 tokens 32768。
  - 最大上下文长度：GLM-4.5 128K，Qwen3.5 256K；最多 200 动作步骤。
  - WebSwarm 设置：`entity_collect` 并行路径数 3，侦察子节点数 2。

## 6. 主要结论与发现
- WebSwarm **在四个基准上均取得最佳或竞争性结果**，优于单智能体 ReAct 和所有多智能体基线。
  - BC-Plus ACC 提升 17.50 点（相对于 ReAct GLM-4.5），超过最强多智能体基线 3.50 点。
  - WideSearch Item-F1 提升 9.76 点，Row-F1 提升 10.91 点。
  - DeepWideSearch Item-F1 提升 11.77 点。
  - GISA 多重指标上均取得最优。
- **优势随任务难度增加而扩大**：在困难子集上提升尤为显著（BC-Plus Hard 子集 ACC 从 0 增至 35.7；WideSearch Hard 子集 Item-F1 从 24.5 增至 55.8）。
- **Web 探测主要提升效率**（减少冗余工具调用），而经验复用主要提升质量（提高准确率）。
- **递归委托和搜索模式的选择是关键**：去除递归委托或强制单一模式均导致性能下降。
- **模型泛化性良好**：在较弱模型（Qwen3-32B）和较强模型（Qwen3.5-35B）上均能带来提升。

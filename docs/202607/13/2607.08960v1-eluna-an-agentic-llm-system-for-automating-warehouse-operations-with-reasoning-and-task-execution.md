## 摘要
仓库操作由标准操作程序（SOP）控制，这些程序编码了复杂的多系统决策逻辑，必须在严格的时间限制下可靠执行。然而，LLM智能体缺乏强制程序合规性的机制，且当完整SOP规范引入上下文过载时会性能下降。我们提出Eluna，一个投产部署的智能体系统，用于可靠执行SOP。Eluna是一个图引导的多智能体框架，将SOP编码为具有渐进式披露的有向无环图，并将独立任务委托给并行子智能体，每个子智能体拥有持久代码执行和实时数据访问。为满足生产延迟和精度需求，我们使用非对称情景蒸馏：通过情景错误记忆改进强教师模型，然后在修正后的轨迹上微调更小的学生模型，去除记忆，内化修正且无推理时间开销。在13个任务的基准测试和两个生产应用中，我们的微调模型匹配或超越其教师，击败所有更大的现成基线，并在工单处理应用中达到94%的专家一致性。

## Abstract
Warehouse operations are governed by Standard Operating Procedures (SOPs) that encode complex, multi-system decision logic, which must be executed reliably under strict time constraints, yet LLM agents lack mechanisms to enforce procedural compliance and degrade under the context overload full SOP specifications introduce. We present Eluna, a production-deployed agentic system for reliable SOP execution. Eluna is a graph-guided, multi-agent framework that encodes SOPs as directed acyclic graphs with progressive disclosure and delegates independent tasks to parallel sub-agents, each with persistent code execution and live data access. To meet production latency and accuracy needs, we use asymmetric episodic distillation where a strong teacher is improved through episodic error memories, then a smaller student is fine-tuned on the corrected trajectories with memory stripped, internalizing corrections without inference-time overhead. On a 13-task benchmark and two production applications, our fine-tuned models match or exceed their teacher, beat all larger off-the-shelf baselines, and reach 94% expert agreement on the ticket processing application.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题**：现代仓库操作依赖标准操作程序（SOP），这些SOP编码了多系统决策逻辑，必须在严格时间限制下可靠执行。然而现有LLM代理（如ReAct、Reflexion等）缺乏强制程序合规的机制，且在完整SOP规范下因**上下文过载**（context overload）而性能严重下降——无论模型规模多大，展示完整图表都会导致模型无法聚焦相关步骤。
- **背景**：当前SOP执行多为人工操作，慢且易出错。虽然LLM具备推理能力，但松散提示无法保证决策路径的确定性。即使SOP增强的多智能体系统（如Flow-of-Action）也面临性能瓶颈。因此，需要一种既能严格遵循程序规范、又能抵抗上下文过载的自动化系统。
- **意义**：Eluna系统首次实现生产级部署的SOP自动化执行，通过图引导结构化推理与精心设计的蒸馏训练，在延迟和精度上同时满足业务需求，证明了**中小模型+结构化训练可以超越大型通用模型**。

## 2. 方法论：核心思想与关键技术细节

- **核心思想**：将SOP编码为**有向无环图（DAG）**，通过**渐进式披露**（progressive disclosure）仅展示可达子图及当前正在评估的节点规范，以此解决上下文过载。采用**分层多智能体架构**，一个主智能体负责图遍历与任务调度，将独立节点评估委托给多个并行子智能体，每个子智能体拥有独立上下文、持久代码解释器（CodeAct）和通过MCP（Model Context Protocol）的实时数据访问，实现上下文隔离与并行执行。
- **关键技术细节**：
  - **图结构SOP表示**：DAG包含五种节点类型（观测、根因、计算、约束、动作），边编码依赖关系（单节点/多节点）。每个节点有参数化规范，评估输出结构化的布尔值、分类或计算值。
  - **渐进式披露**：两级策略——首先BFS从触发节点返回仅可达子图（节点类型、边关系、简短描述）；随后仅在开始评估某节点时才获取其完整规范，将工作上下文限制为单个节点一次。
  - **并行子代理**：主智能体在单回合内发出多个委托调用并行执行，子代理返回结构化结论，主智能体上下文仅增长一个摘要行。子代理会话持久化，但不可再创建子代理，从而避免嵌套膨胀。
  - **技能包装（Skill Packaging）**：每个工作流程打包成一个自包含的技能（决策图、MCP工具、节点规范、CodeAct函数、执行指令），运行时按需加载，使框架保持用例无关，添加新工作流只需编写技能。
  - **工具生态系统**：持久代码解释器（保留状态）、TodoList（驱动遍历）、MCP工具（实时数据查询）、Agentic RAG（按需知识检索）、记忆（跨调用持久化，防重复调查）。
- **训练策略**：
  - **非对称情景蒸馏（Asymmetric Episodic Distillation）**：
    1. 教师模型（GLM-4.7，355B）在完整框架内生成轨迹；
    2. 通过情景学习迭代改进教师轨迹：对失败案例，单独LLM分析错误并生成候选记忆条目，合并冗余后注入提示，教师重新运行，迭代N轮直至收敛（无权重更新）；
    3. 对教师轨迹进行按轮分解（turn-level decomposition），每轮将历史对话作为输入，目标为该轮推理与动作，并剥离先前推理；
    4. 采用拒绝采样（rejection sampling）和LLM-as-judge过滤，仅保留完全正确的轨迹。
    5. 使用改进后的轨迹对学生模型（Qwen3-32B或GLM-4.5-Air）进行监督微调（LoRA），**去除情景记忆**，让学生将修正内化到权重中，推理时无记忆依赖。

## 6. 论文的主要结论与发现

1. **蒸馏胜过规模**：在Eluna图引导框架中，微调的32B模型匹配甚至超越355B教师，并击败所有现成模型（无论大小）。现成模型性能出现平台期，说明通用训练不足以应对程序合规任务。
2. **非对称情景蒸馏是关键**：将情景记忆用于教师轨迹改进，但去除学生训练中的记忆，比在推理时依赖记忆更鲁棒且无延迟成本；若学生训练时暴露记忆但推理时移除，性能灾难性下降（Task 5降53%，Task 10降40%）。
3. **剩余错误根因不在推理**：生产部署中的错误主要来源于SOP规范缺失和上游数据质量问题，而非代理推理错误。
4. **实际业务收益**：机器人输送应用中诊断速度提升300%，端到端延迟降低54%–55%，SOP遵循率近乎完美，已处理约8000次触发。工单处理自动化率达94.4%专家一致性，将80%需手动处理的任务中的20%降至仅2.2%需人工介入。

## 7. 优点

- **方法创新性**：将SOP编码为DAG并采用渐进式披露，创新性地解决了LLM在长程序约束下的上下文过载问题；并行子代理设计兼顾上下文隔离与执行效率。
- **训练设计巧妙**：非对称情景蒸馏充分利用大型教师模型的自我改进能力，同时保证学生在线推理时无额外负担，实现精度与延迟双赢。
- **实际部署验证**：系统已在亚马逊生产环境部署，处理真实业务数据，并经过专家评审，证实了方法的可行性和鲁棒性。
- **落地导向**：明确以生产延迟SLA为约束设计模型选择与训练策略（106B作为可部署基线，355B仅用于工单处理的长间隔场景），并量化了业务指标改进。
- **通用性设计**：技能打包机制使添加新工作流无需修改框架，可扩展性强。

## 8. 不足与局限

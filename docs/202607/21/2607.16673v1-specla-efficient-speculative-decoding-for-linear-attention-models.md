## 摘要
线性注意力模型将不断增长的KV缓存替换为循环状态，但自回归解码仍然每次只读取、更新和写入一个令牌的状态。推测解码可以通过在一次目标传递中验证多个草稿令牌来降低这一成本，然而现有的推测系统是为Transformer KV缓存设计的。对于有状态的线性注意力目标，验证必须遵循跨链和分支的循环依赖，接受时必须仅更新已接受的状态轨迹，并且草稿生成器必须避免提交浪费有状态验证工作的候选方案。本文提出了SpecLA，一种针对有状态线性注意力模型的推测解码运行时系统。SpecLA使用拓扑感知内核验证链和树，存储验证过程中产生的紧凑因子以恢复已接受的状态，并采用置信剪枝以及目标对齐的EAGLE风格草稿生成器，向验证器提供有用的候选方案。在配备NVIDIA H100且使用公开GDN-1.3B目标模型的情况下，SpecLA相比自回归解码实现了高达1.70倍的端到端加速。

## Abstract
Linear-attention models replace the growing KV cache with recurrent states, but autoregressive decoding still reads, updates, and writes these states one token at a time. Speculative decoding can reduce this cost by verifying several draft tokens in one target pass, yet existing speculative systems are designed for Transformer KV caches. For stateful linear-attention targets, verification must follow recurrent dependencies across chains and branches, acceptance must update only the accepted state trajectory, and the drafter must avoid submitting candidates that waste stateful verification work. This paper presents SpecLA, a speculative decoding runtime for stateful linear-attention models. SpecLA verifies chains and trees with topology-aware kernels, stores compact factors produced during verification to recover accepted states, and uses confidence pruning plus a target-aligned EAGLE-style drafter to feed useful candidates to the verifier. On an NVIDIA H100 with a public GDN-1.3B target, SpecLA achieves up to 1.70x end-to-end speedup over autoregressive decoding.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：线性注意力模型（如 Gated DeltaNet）使用固定的循环状态（recurrent state）替代了传统 Transformer 不断增长的 KV 缓存。
    - **背景挑战**：尽管减少了内存占用，但该模型的自回归解码过程仍然需要逐 token 读取、更新和写入这个密集的循环状态，导致解码过程受限于内存带宽，成为性能瓶颈。
- **现有方法的局限**：
    - **推测解码（Speculative Decoding）** 是加速 Transformer 解码的有效方法，但其验证和接受机制高度依赖 KV 缓存的“后缀截断”操作。
    - 直接将其应用于线性注意力模型会失败：
        - **验证低效**：使用解码内核进行重放（Replay）仍然带来每 token 的循环状态往返开销；直接使用预填充内核（Prefill）在短推测窗口下无法摊还初始化成本，且无法处理分支候选状态（树形推测）。
        - **状态恢复困难**：被拒绝的 token 无法像 Transformer 那样简单截断，需要复杂且昂贵的状态回滚机制。
- **研究动机**：如何为具有循环状态的线性注意力模型设计一套高效的推测解码系统，使其能够克服上述挑战，实现真正的端到端加速。

### 2. 论文提出的方法论：核心思想与关键技术

论文提出了 **SpecLA** 运行时系统，其核心思想是“为有状态的线性注意力模型量身定制推测解码的每一环节”，具体包含三个层面的关键技术：

1.  **拓扑感知验证（Topology-Aware Verification）**：
    - **核心思想**：根据提交的候选序列拓扑（链或树）动态选择最优的验证策略。
    - **链状结构**：使用**状态驻留串行验证（State-Resident Serial Verification）**。改变执行顺序为“layer-major”，在同一层内连续处理多个候选 token，让循环状态在芯片上的 SRAM 中驻留，避免反复读写 HBM。并通过 **V 维度平铺** 解决状态过大无法完全放入 SRAM 的问题。
    - **树状结构**：使用**链分解混合验证（Chain-Decomposed Hybrid Verification）**。将提交的推测树分解为若干个相互无依赖的“重链”（Heavy Chain），每个链内部使用低开销的状态驻留串行验证；不同链之间则可以并行验证，达到效率与并行度的平衡。
    - 另外提出了一种理论上更精确但开销更大的**树掩码并行验证（Tree-Masked Parallel Verification）** 作为对比，用于说明混合方法的优势。

2.  **接受状态管理（Speculative State Management）**：
    - **核心思想**：验证过程会产生更新状态所需的“因子（factors）”（如 key, value 向量），这些因子体积远小于完整的循环状态。
    - **因子缓冲（Factor Buffering）**：在验证阶段，不存储全量状态的快照，也不在事后重放（Replay）已接受 token，而是缓存每个候选 token 产生的紧凑的**状态更新因子**。在接受后，仅对这些因子做轻量级的“Delta 构造（DeltaConstruct）”来更新状态，消除冗余计算。
    - **延迟状态更新（Delayed State Update）**：不立即将接受后的状态更新写回 HBM，而是将其作为“待处理因子缓冲区”保留。在下一轮验证开始时，通过**融合的更新-验证内核（Fused Update-Verify Kernel）**，在加载状态到 SRAM 的同时应用这些因子，从而消除一次独立的状态写回和读取操作。

3.  **目标对齐的草稿集成（Target-Aligned Draft Integration）**：
    - **核心思想**：草稿生成器的输出需要与线性注意力目标模型的执行路径和状态特征对齐。
    - **置信度引导剪枝**：草稿生成后，根据候选路径的累积对数概率进行剪枝，过滤掉低置信度的分支，减少无用的验证工作。
    - **目标对齐的 EAGLE 风格草稿生成器**：训练一个 EAGLE 风格的草稿层，其输入特征提取自目标线性注意力模型本身的循环状态，使其“学会”在正确上下文中提出候选项。但草稿内部计算仍保留 Transformer 风格，因为在短草稿窗口下，Transformer 轻量且易于实现。

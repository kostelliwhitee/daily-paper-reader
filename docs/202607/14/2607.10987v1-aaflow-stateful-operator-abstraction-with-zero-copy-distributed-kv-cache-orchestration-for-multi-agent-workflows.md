## 摘要
多智能体LLM系统日益集成检索、规划和推理功能，但本质上仍以文本为中心，要求智能体通过昂贵的预填充重复计算共享上下文。尽管已知KV缓存管理可以加速单请求推理，但它通常局限于本地服务范围。我们提出AAFLOW+，一种智能体工作流算子的状态化扩展，将KV缓存作为分布式系统的一等公民对象。AAFLOW+将流程构建为通信感知图，同时优化数据、提示和可重用的模型状态。它还提供了KV物化、传输、分支、组合和驱逐等算子。其运行时支持零拷贝、传输感知的执行，使智能体无需重新计算即可重用长上下文。基于经验硬件微基准参数化的分析成本模型，AAFLOW+可将TTFT降低高达50.2倍，在16智能体规模下将多智能体计算成本降低高达7.63倍，KV内存减少1.72-6.10倍，吞吐量提升超过7.74倍。结果表明，在中高带宽网络上，KV传输优于重新计算，通过替换文本传递，KV状态共享极大地提高了多智能体LLM系统的效率。

## Abstract
Multi-agent LLM systems increasingly integrate retrieval, planning, and reasoning, but remain fundamentally text-centric, requiring agents to repeatedly recompute shared context through expensive prefill. Although single-request inference is known to be accelerated by KV-cache management, it is usually restricted to local serving scopes. We introduce AAFLOW+, a stateful extension of agentic workflow operators that makes KV cache a first-class distributed systems object. AAFLOW+ builds processes into communication-aware graphs that concurrently optimize data, prompts, and reusable model state. It also provides operators for KV materialization, transfer, fork, composition, and eviction. Its runtime enables zero-copy, transfer-aware execution, allowing agents to reuse long context without recomputation. AAFLOW+ reduces TTFT by up to 50.2x, achieves up to 7.63x reduced multi-agent compute cost at 16-agent scale, reduces KV memory by 1.72-6.10x, and increases throughput by more than 7.74x, based on an analytical cost model parameterized by empirical hardware microbenchmarks. The results demonstrate that KV transmission outperforms recomputation on networks with moderate to high bandwidth, making sure KV-state sharing greatly increases efficiency in multi-agent LLM systems by replacing text passing.

- **核心问题**：现有系统（如vLLM、SGLang）虽对KV缓存进行了局部优化（如分页管理、前缀共享），但**缺乏将KV缓存作为跨智能体工作流的一等分布式对象**进行抽象、传输、分支和复用的能力。多智能体推理、树状思维、协作RAG等场景中共享上下文被多次重建，造成时间到首令牌（TTFT）高、计算冗余、框架开销大、内存压力大。
- **整体含义**：本文提出将KV缓存从本地推理优化提升为**分布式工作流状态对象**，通过显式的状态算子（物化、传输、分支、组合、驱逐）实现跨智能体零拷贝复用，从根本上解决文本传递带来的重复计算问题。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 在AAFLOW（通信感知DAG执行框架）基础上扩展为**状态化算子抽象**，令KV缓存成为可调度、可传输、可复用的分布式系统对象。
- 定义**状态化算子族**：包括KV物化（materialize）、传输（transfer）、分支（fork）、受限组合（merge）、驱逐（eviction）。
- 运行时实现**零拷贝、传输感知的执行**，通过显式元数据和Arrow格式分离元数据与张量缓冲区，利用UCX/RDMA/MPI实现高效跨节点传输。
- 编译器可自动选择文本传递、KV复用、状态传输或重计算，以最小化总延迟和内存开销。

### 关键技术细节
- **状态化算子定义**：`Op_s_i = (I_i, O_i, S_in_i, S_out_i, f_i, P_i, σ_i)`，其中`S_in_i, S_out_i`为输入输出状态对象，`σ_i`为状态策略（传输、别名、分支、合并、固定、驱逐或重计算）。
- **KV状态对象**：`S_KV = (M, Θ, B, Π, Λ, Γ)`，包含模型标识、配置、KV块集合、位置元数据、谱系元数据、放置与所有权信息。块表示为`b_j = (K_j, V_j, ℓ_j, r_j, d_j)`。
- **兼容性谓词**：`Compat(S_a,S_b)`要求模型、配置相同且位置兼容，否则回退到文本重计算。
- **状态算子语义**：
  - 物化：`Op_kv_materialize(x, M) → S_KV`
  - 传输：`Op_kv_transfer(S_KV, n_a, n_b) → S'_KV`
  - 分支：`Op_kv_fork(S_KV, k) → {S^(1)_KV, ..., S^(k)_KV}`，初始共享物理块，写时复制。
  - 受限合并：仅允许前缀兼容拼接或通过模型调用的文本级归约，避免张量级混合破坏位置编码。
- **成本模型**：当转移成本小于重计算成本时选择状态传输。`T_transfer = |KV|/BW + δ`，`T_recompute = T_prefill`。分支工作流中，文本总成本为`k·T_prefill + sum_decode`，状态化总成本为`T_prefill + T_fork + sum_decode`，节省`(k-1)T_prefill - T_fork`。
- **生命周期管理**：状态先创建、注册、激活，后分支、传输、恢复，最终可驱逐。驱逐采用启发式评分（基于重用频率、大小、年龄）。
- **状态感知调度器**：对每个操作评估传输或重计算，并优化总延迟与内存的权衡。

1. **TTFT显著降低**：AAFLOW+在Mistral上平均TTFT为0.041s，比AAFLOW-text（2.057s）快**50.2倍**，比DistServe（0.123s）快3.0倍；在Llama3上平均0.030s，比DistServe（0.052s）快1.7倍。差距在大上下文时更为突出。

6. **KV共享替换文本传递**是提高多智能体LLM系统效率的关键机制，尤其适用于中等高带宽网络环境。

- **适用范围受限**：仅针对共享前缀的多智能体工作流有效，不适用于独立无状态的短提示场景。若输出长度很大（如数千token），解码将占主导，TTFT优势减弱。

- **没有实现真正的端到端应用**：缺乏如AutoGen、LangGraph等框架中的完整多智能体对话或任务执行的直接对比。
